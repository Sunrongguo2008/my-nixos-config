## 配置文件

mangowm 使用简单的配置文件格式。默认情况下，它会在 `~/.config/mango/` 中查找配置文件。

1. **定位默认配置**

   系统提供了一个备用配置文件 `/etc/mango/config.conf`。你可以将其作为参考。

2. **创建用户配置**

   将默认配置复制到本地配置目录以开始自定义。

   ```bash
   mkdir -p ~/.config/mango
   cp /etc/mango/config.conf ~/.config/mango/config.conf
   ```

3. **使用自定义配置启动（可选）**

   如果你希望将配置保存在其他位置，可以使用 `-c` 标志启动 mango。

   ```bash
   mango -c /path/to/your_config.conf
   ```

### 子配置

为了保持配置文件的组织结构清晰，你可以将配置拆分为多个文件，并使用 `source` 关键字导入它们。

```ini
# 从单独的文件导入键位绑定
source=~/.config/mango/bind.conf

# 相对路径也适用
source=./theme.conf

# 可选：如果文件不存在则忽略（适用于共享配置）
source-optional=~/.config/mango/optional.conf
```

### 验证配置

你可以在不启动 mangowm 的情况下检查配置是否有错误：

```bash
mango -p /path/to/config.conf
```

可与 `source-optional` 配合使用，适用于不同 setups 的共享配置。

## 环境变量

你可以在配置文件中直接定义环境变量。这些变量会在窗口管理器完全初始化之前设置。

> **警告：** 此处定义的环境变量会在每次重新加载配置时被**重置**。

```ini
env=GTK_THEME,Adwaita:dark
env=XCURSOR_SIZE,24
```

## 自启动

mangowm 可以在启动时自动运行命令或脚本。有两种执行模式：

| 命令        | 行为                              | 使用场景                   |
| :---------- | :-------------------------------- | :------------------------- |
| `exec-once` | 仅在 mangowm 启动时运行**一次**。 | 状态栏、壁纸、通知守护进程 |
| `exec`      | 每次重新加载配置时运行。          | 需要刷新设置的脚本         |

### 示例设置

```ini
# 启动状态栏一次
exec-once=waybar

# 设置壁纸
exec-once=swaybg -i ~/.config/mango/wallpaper/room.png

# 在配置更改时重新加载自定义脚本
exec=bash ~/.config/mango/reload-settings.sh
```
