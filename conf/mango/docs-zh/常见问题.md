### 如何用鼠标排列平铺窗口？

你可以在配置中启用 `drag_tile_to_tile` 选项。这允许你将一个平铺窗口拖放到另一个窗口上来交换它们的位置。

```ini
drag_tile_to_tile=1
```

---

### 为什么我的背景模糊或者模糊效果看起来不正常？

模糊应用于窗口的透明区域。要完全禁用它，设置 `blur=0`。

如果你遇到**模糊性能问题**，请确保 `blur_optimized=1`（默认值）。这会将壁纸缓存为模糊背景，对 GPU 的消耗要小得多：

```ini
blur_optimized=1
```

---

### 模糊显示的是壁纸而不是真实的背景内容

这是 `blur_optimized=1`（默认值）时的预期行为。优化器会缓存壁纸以减少 GPU 负载 —— 窗口会根据壁纸而不是实际堆叠在下方的内容进行模糊。

如果你希望模糊针对真实的背景（即显示窗口后面实际的内容），设置：

```ini
blur_optimized=0
```

> **警告：** 禁用 `blur_optimized` 会显著增加 GPU 消耗，并可能导致渲染卡顿，尤其是在低端硬件上。

### 我的游戏卡顿或掉帧

尝试启用 **SyncObj** 时间线支持和**自适应同步**（VRR），如果你的显示器支持的话。

```ini
syncobj_enable=1
adaptive_sync=1
```

---

### 我的游戏输入延迟很高

你可以启用**撕裂**（类似于关闭 VSync）。

首先，全局启用：

```ini
allow_tearing=1
```

然后为你的特定游戏强制启用：

```ini
windowrule=force_tearing:1,title:Counter-Strike 2
```

> **警告：** 某些显卡需要在启动 mango 之前设置 `WLR_DRM_NO_ATOMIC=1` 才能使撕裂生效。将其添加到 `/etc/environment` 并重启，或者使用 `WLR_DRM_NO_ATOMIC=1 mango` 启动 mango。详见 [显示器 — 撕裂游戏模式](/docs/configuration/monitors#tearing-game-mode) 了解详细信息。

### 如何在 spawn 命令中使用管道 `|`？

标准的 `spawn` 命令不支持 shell 管道。你必须改用 `spawn_shell`。

```ini
bind=SUPER,P,spawn_shell,echo "hello" | rofi -dmenu
```

### 某些键盘组合在我的键盘布局上不起作用。

`bind` 会自动将 keysym 转换为 keycode，这兼容大多数布局，但有时可能不够精确。如果某个键组合没有触发，请使用**keycode**而不是键名。

运行 `wev` 并按下该键以查找其 keycode，然后在 bind 中使用它：

```ini
# 而不是：
bind=ALT,q,killclient

# 使用 keycode（例如：code:24 = 大多数布局上的 q）：
bind=ALT,code:24,killclient
```

你也可以使用 `binds`（`s` 标志）来通过 keysym 而不是 keycode 进行匹配：

```ini
binds=ALT,q,killclient
```
