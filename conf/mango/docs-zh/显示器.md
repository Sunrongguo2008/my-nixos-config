## 显示器规则

你可以使用 `monitorrule` 关键字单独配置每个显示输出。

**语法：**

```ini
monitorrule=name:Values,Parameter:Values,Parameter:Values
```

> **信息：** 如果设置了任何匹配字段（`name`、`make`、`model`、`serial`），则**所有**设置的字段都必须匹配才被视为匹配。使用 `wlr-randr` 获取显示器的名称、制造商、型号和序列号。

### 参数

| 参数      | 类型   | 值           | 描述                                                  |
| :-------- | :----- | :----------- | :---------------------------------------------------- |
| `name`    | 字符串 | 任意         | 按显示器名称匹配（支持正则表达式）                    |
| `make`    | 字符串 | 任意         | 按显示器制造商匹配                                    |
| `model`   | 字符串 | 任意         | 按显示器型号匹配                                      |
| `serial`  | 字符串 | 任意         | 按显示器序列号匹配                                    |
| `width`   | 整数   | 0-9999       | 显示器宽度                                            |
| `height`  | 整数   | 0-9999       | 显示器高度                                            |
| `refresh` | 浮点数 | 0.001-9999.0 | 显示器刷新率                                          |
| `x`       | 整数   | 0-99999      | X 位置                                                |
| `y`       | 整数   | 0-99999      | Y 位置                                                |
| `scale`   | 浮点数 | 0.01-100.0   | 显示器缩放                                            |
| `vrr`     | 整数   | 0, 1         | 启用可变刷新率                                        |
| `rr`      | 整数   | 0-7          | 显示器变换                                            |
| `custom`  | 整数   | 0, 1         | 启用自定义模式（并非所有显示器都支持 — 可能导致黑屏） |

### 变换值

| 值   | 旋转               |
| :--- | :----------------- |
| `0`  | 无变换             |
| `1`  | 逆时针 90°         |
| `2`  | 逆时针 180°        |
| `3`  | 逆时针 270°        |
| `4`  | 垂直翻转 180°      |
| `5`  | 翻转 + 逆时针 90°  |
| `6`  | 翻转 + 逆时针 180° |
| `7`  | 翻转 + 逆时针 270° |

> **关键：** 如果你使用 XWayland 应用程序，**切勿为显示器位置使用负坐标**。这是一个已知的 XWayland bug，会导致点击事件失效。始终从 `0,0` 开始排列显示器并扩展到正坐标。

### 示例

```ini
# 笔记本显示屏：1080p, 60Hz, 位于原点
monitorrule=name:eDP-1,width:1920,height:1080,refresh:60,x:0,y:10

# 按制造商和型号匹配而不是名称
monitorrule=make:Chimei Innolux Corporation,model:0x15F5,width:1920,height:1080,refresh:60,x:0,y:0

# 使用模式匹配的虚拟显示器
monitorrule=name:HEADLESS-.*,width:1920,height:1080,refresh:60,x:1926,y:0,scale:1,rr:0,vrr:0
```

## 显示器规格格式

多个命令（`focusmon`、`tagmon`、`disable_monitor`、`enable_monitor`、`toggle_monitor`、`viewcrossmon`、`tagcrossmon`）接受 **monitor_spec** 字符串来标识显示器。

**格式：**

```text
name:xxx&&make:xxx&&model:xxx&&serial:xxx
```

- 任何字段都可以省略，没有顺序要求。
- 如果所有字段都省略，则字符串直接被视为显示器名称（例如 `eDP-1`）。
- 使用 `wlr-randr` 查找显示器的名称、制造商、型号和序列号。

**示例：**

```bash
# 按名称（简写）
mmsg -d toggle_monitor,eDP-1

# 按制造商和型号
mmsg -d toggle_monitor,make:Chimei Innolux Corporation&&model:0x15F5

# 按序列号
mmsg -d toggle_monitor,serial:12345678
```

## 撕裂（游戏模式）

撕裂允许游戏绕过合成器的 VSync 以降低延迟。

| 设置            | 默认值 | 描述                                                    |
| :-------------- | :----- | :------------------------------------------------------ |
| `allow_tearing` | `0`    | 全局撕裂控制：`0`（禁用）、`1`（启用）、`2`（仅全屏）。 |

### 配置

**全局启用：**

```ini
allow_tearing=1
```

**按窗口启用：**

使用窗口规则为特定游戏强制撕裂。

```ini
windowrule=force_tearing:1,title:vkcube
```

### 撕裂行为矩阵

| `force_tearing` \ `allow_tearing` | DISABLED (0) | ENABLED (1)       | FULLSCREEN_ONLY (2)     |
| :-------------------------------- | :----------- | :---------------- | :---------------------- |
| **UNSPECIFIED** (0)               | 不允许       | 遵循 tearing_hint | 仅全屏遵循 tearing_hint |
| **ENABLED** (1)                   | 不允许       | 允许              | 仅全屏允许              |
| **DISABLED** (2)                  | 不允许       | 不允许            | 不允许                  |

### 显卡兼容性

> **警告：** 某些显卡需要在启动 mango 之前设置 `WLR_DRM_NO_ATOMIC` 环境变量才能成功启用撕裂。

将其添加到 `/etc/environment` 并重启：

```bash
WLR_DRM_NO_ATOMIC=1
```

或使用环境变量运行 mango：

```bash
WLR_DRM_NO_ATOMIC=1 mango
```

## GPU 兼容性

如果 mango 无法正确显示或显示黑屏，尝试选择特定的 GPU：

```bash
# 使用单个 GPU
WLR_DRM_DEVICES=/dev/dri/card1 mango

# 使用多个 GPU
WLR_DRM_DEVICES=/dev/dri/card0:/dev/dri/card1 mango
```

某些 GPU 与 `syncobj_enable=1` 存在兼容性问题 — 它可能导致使用 syncobj 的应用程序（如 `kitty`）崩溃。在 `/etc/environment` 中设置 `WLR_DRM_NO_ATOMIC=1` 并重启以解决此问题。

## 电源管理

你可以使用 `mmsg` IPC 工具控制显示器电源。

```bash
# 关闭
mmsg -d disable_monitor,eDP-1

# 开启
mmsg -d enable_monitor,eDP-1

# 切换
mmsg -d toggle_monitor,eDP-1
```

你也可以使用 `wlr-randr` 进行显示器管理：

```bash
# 关闭显示器
wlr-randr --output eDP-1 --off

# 开启显示器
wlr-randr --output eDP-1 --on

# 显示所有显示器
wlr-randr
```

## 屏幕缩放

### 不使用全局缩放（推荐）

- 如果你不使用 XWayland 应用程序，你可以使用显示器规则或 `wlr-randr` 设置全局显示器缩放。
- 如果你正在使用 XWayland 应用程序，不建议设置全局显示器缩放。

你可以这样设置缩放，例如使用 1.4 因子。

**依赖：**

```bash
yay -S xorg-xrdb
yay -S xwayland-satellite
```

**在配置文件中：**

```ini
env=QT_AUTO_SCREEN_SCALE_FACTOR,1
env=QT_WAYLAND_FORCE_DPI,140
```

**在自启动中：**

```bash
echo "Xft.dpi: 140" | xrdb -merge
gsettings set org.gnome.desktop.interface text-scaling-factor 1.4
```

**为 XWayland 编辑自启动：**

```bash
# 启动 xwayland
/usr/sbin/xwayland-satellite :11 &
# 为 xwayland 应用 1.4 缩放
sleep 0.5s && echo "Xft.dpi: 140" | xrdb -merge
```

### 使用 xwayland-satellite 防止 XWayland 应用程序模糊

如果你使用分数缩放，你可以使用 `xwayland-satellite` 自动缩放 XWayland 应用程序以防止模糊，例如使用 1.4 缩放。

**依赖：**

```bash
yay -S xwayland-satellite
```

**在配置文件中：**

```ini
env=DISPLAY,:2
exec=xwayland-satellite :2
monitorrule=name:eDP-1,width:1920,height:1080,refresh:60,x:0,y:0,scale:1.4,vrr:0,rr:0
```

> **警告：** 使用不同于 `:1` 的 `DISPLAY` 值以避免与 mangowm 冲突。

## 虚拟显示器

你可以通过 IPC 命令创建和管理虚拟显示器：

```bash
# 创建虚拟输出
mmsg -d create_virtual_output

# 销毁所有虚拟输出
mmsg -d destroy_all_virtual_output
```

你可以使用 `wlr-randr` 配置虚拟显示器：

```bash
# 显示所有显示器
wlr-randr

# 配置虚拟显示器
wlr-randr --output HEADLESS-1 --pos 1921,0 --scale 1 --custom-mode 1920x1080@60Hz
```

虚拟显示器可用于与 [Sunshine](https://github.com/LizardByte/Sunshine) 和 [Moonlight](https://github.com/moonlight-stream/moonlight-android) 等工具进行屏幕共享，允许其他设备作为扩展显示器使用。
